# 回测与评估层 — 技术规格

> 配套主文档 [架构设计方案-完善版-v2.md](架构设计方案-完善版-v2.md)。
> **这是此前文档缺失的关键能力**：没有可信回测/纸面交易，任何策略都不应进入实盘。
> 本层是路线图 P4 实盘灰度的**前置门槛**（见主文档 §10）。

---

## 1. 目标与原则

- **决策前可验证**：每个策略上线前，须在历史数据上跑出可信的收益/风险/校准指标。
- **真实成本与机制**：回测必须含费用、Gas、滑点、部分成交、腿险、以及 **UMA 结算延迟/资金锁定**
  （见 [链上机制与结算-技术规格.md](链上机制与结算-技术规格.md) §3）。
- **防前视偏差（look-ahead）**：信号只能用 `event_ts ≤ 决策时刻` 的数据；杜绝用未来信息。
- **可复现**：固定随机种子、数据快照版本化、配置即代码。

---

## 2. 历史数据源

| 用途 | 数据源 | 说明 |
| --- | --- | --- |
| Polymarket 成交/订单簿历史 | Data API / 官方 subgraph（The Graph） | 历史价格、成交、市场元数据与结算结果 |
| 市场结算标签 | 链上 resolve 结果 / Data API | 已结算市场的真实结果，用于校准与命中率 |
| 钱包 PnL（策略⑥） | subgraph / Data API 按地址聚合 | 构建 `WalletProfile`（仅公开数据，合规见主文档 §9.7） |
| 加密锚点 | Binance klines / 历史 tick | 策略④ 公允价回放 |
| 宏观/基本面 | OpenBB（fred/bls/sec） | 事件型先验回测 |

- 数据落**版本化快照**（不可变），回测引用快照 id，保证可复现。

---

## 3. 成交（fill）与成本模型

回测撮合不可假设"信号价即成交价"：

- **滑点/深度**：按当时订单簿深度模拟吃单滑点；薄市场按 `liquidity.py`（数据层 §6.2）口径限制可成交量。
- **部分成交**：多腿策略（①②）模拟单腿成交、另一腿落空 → 触发 merge/convert 兜底成本
  （见链上机制文档 §1.1/§2.1）。
- **费用 + Gas**：按当期费率与 Polygon Gas 估算计入每笔（占位参数，待复核）。
- **延迟**：对延时套利④，按 NFR §11.1 的端到端延迟与 `Δ_max` 模拟"信号到下单"的时间损耗与价格漂移。
- **结算延迟与资金锁定**：按 UMA 时间线（~2h / 争议 4–6 天）锁定资金，计入资金占用与机会成本。

## 4. 验证方法

- **Walk-forward**：滚动窗口"样本内调参 → 样本外检验"，避免一次性全样本过拟合。
- **样本外/分域**：按领域（politics/crypto/sports/macro）与时间分层分别评估，防止单域偶然。
- **稳健性**：对关键参数（θ、κ、Δ_max、波动率 λ）做敏感性扫描；对冲洗盘样本（剔除 `wash_suspect`）。
- **基准对照**：与"不交易""随机下注""市价跟随"等基准比较，确认 Alpha 真实存在。

## 5. 指标与 PnL 归因

| 类别 | 指标 |
| --- | --- |
| 收益 | 净收益、年化、复合增长；扣全部成本后口径 |
| 风险 | 最大回撤、波动、**VaR / CVaR（尾部，重点用于策略⑤）**、夏普/索提诺 |
| 胜率结构 | 命中率、盈亏比、**收益偏度（识别负偏 steamroller）** |
| 校准 | Brier score、可靠性曲线（喂主文档 §4.4 校准监控与 §9.4 衰减判定） |
| 归因 | **按策略 / 市场 / 领域拆解 PnL**，定位 Alpha 来源与衰减 |

- 校准与命中率的标签来自 §2 的已结算市场，闭环回流推理层与风控。

---

## 6. 纸面交易（paper-trading）

- 复用生产数据/信号/编排链路，但**执行器替换为模拟撮合**（不下真单、不签名、不上链）。
- 与回测共用成本/成交模型，作为"实盘前最后一道关"，验证端到端延迟、状态机、风控触发与对账逻辑。
- 路线图：P3 编排层即以 paper-trading 跑通；P4 才接小资金实盘（主文档 §10）。

## 7. 模块划分（建议）

```
backtest/
  data/            # 历史快照拉取与版本化（subgraph / Data API / Binance / OpenBB）
  fills/           # 成交/滑点/部分成交/腿险/结算延迟模型
  engine/          # 事件驱动回放引擎（防前视）
  walkforward/     # 滚动样本内外切分与参数扫描
  metrics/         # 收益/风险/校准/PnL 归因
  paper/           # 纸面交易（模拟执行器）
```

> 复用数据层 pydantic 契约与信号模块；执行器在回测/纸面用模拟实现替换生产实现（依赖倒置）。
