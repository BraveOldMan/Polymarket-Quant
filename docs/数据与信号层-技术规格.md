# 数据与信号层 — 技术规格

> 配套主文档：[架构设计方案-完善版-v2.md](架构设计方案-完善版-v2.md) §6。
> 本规格聚焦**第一阶段最小闭环**：采集 → Schema 校验 → 信号 → 经 MCP 注入推理层。
> 本文给出**可执行蓝图与接口契约**，示例代码仅为契约示意，**非生产实现**。

---

## 1. 设计原则（落实 CLAUDE.md 约束）

1. **校验边界**：任何外部数据先经 `pydantic v2` 模型校验，**校验失败即拒绝 + 告警**，绝不进入推理层。
2. **内容字段默认不可信（注入防御）**：Schema 校验只防**格式**。市场标题/规则、新闻、SEC、链上 memo 等
   **自由文本**在注入下游 LLM 前须做 **Prompt 注入防御**：内容与指令分离、转义分隔、注入特征检测；
   LLM 不可由内容驱动触发任何下单/转账（见主文档 §3 硬边界#1、§11.5）。
3. **模块化热替换**：每个数据源是独立 collector，遵循统一 `Collector` 协议，可单独启停/替换。
4. **归一化**：所有源的原始数据先映射为统一的内部事件模型，下游只依赖内部模型，不依赖第三方原始格式。
5. **时间口径统一**：所有事件带 `event_ts`（数据本身时间）与 `recv_ts`（本地接收时间，单调时钟），
   延时套利的时间对齐依赖二者。
6. **只读优先**：第一阶段不接触下单路径，输出信号即终点。

    binance.py         # python-binance / 原生 WS：BTC/ETH 现货+合约
    deribit.py         # OpenBB deribit provider：期权 IV/隐含分布（策略④ 公允价升级，主文档 §4.2a）
    polygon_chain.py   # web3.py + Polygon RPC：鲸鱼地址/USDC 转账

  schemas/             # pydantic v2 数据契约（见 §4）

  bus/                 # 内部事件总线（asyncio.Queue / Redis Streams）
  signals/
    fair_value.py      # Binance → 公允价（策略④ 延时套利）
    latency_arb.py     # 公允价 vs Polymarket 隐含概率 → 价差信号（策略④）
    structural.py      # 单市场 YES+NO 偏离 1 的结构性错价（策略①）
    multi_condition.py # 互斥市场组 ΣYesᵢ 偏离 1（策略②）
    semantic_map.py    # LLM 跨市场语义等价映射 → 脱钩价差（策略③）
    time_decay.py      # 临到期极端目标筛选 → 卖 No 时间价值（策略⑤，高风险）
    smart_money.py     # 链上钱包画像/评分 → 跟单信号（策略⑥，涉合规）
    wash_filter.py     # 洗盘/自成交检测 → 标记并剔除可交易集
    liquidity.py       # 出场流动性预检（流动性陷阱防护）
    whale_alert.py     # 链上大额转账预警
  mcp_server/          # 官方 mcp SDK：暴露 Resources/Tools 给 Claude
  config/              # 阈值、地址白名单、symbol 映射（热加载）

## 4. 接口契约（pydantic v2 模型示意）

> 字段、单位、时间戳口径如下；类型仅示意，落地时补全校验器（范围、枚举、非负等）。

```python
from datetime import datetime
from decimal import Decimal
from enum import Enum
from pydantic import BaseModel, Field

class Side(str, Enum):
    YES = "YES"
    NO = "NO"

class Market(BaseModel):
    """Polymarket 市场元数据（来源 Gamma API）。"""
    condition_id: str
    question: str
    token_id_yes: str
    token_id_no: str
    category: str                      # politics / crypto / sports / ...
    end_date: datetime | None
    active: bool

class OrderBookLevel(BaseModel):
    price: Decimal = Field(ge=0, le=1)  # 概率价 ∈ [0,1]
    size: Decimal = Field(ge=0)

class OrderBookSnapshot(BaseModel):
    """Polymarket 某 token 的订单簿快照。"""
    token_id: str
    bids: list[OrderBookLevel]
    asks: list[OrderBookLevel]
    event_ts: datetime                  # 交易所/数据源时间
    recv_ts: datetime                   # 本地接收时间（单调时钟换算）

    @property
    def mid(self) -> Decimal:           # 隐含概率 = 中间价
        ...

class PriceTick(BaseModel):
    """Binance 行情 tick（公允价锚点输入）。"""
    symbol: str                         # BTCUSDT / ETHUSDT ...
    market_type: str                    # spot / perp
    price: Decimal = Field(gt=0)
    event_ts: datetime
    recv_ts: datetime

class ChainEvent(BaseModel):
    """Polygon 链上事件（鲸鱼/USDC 转账）。"""
    tx_hash: str
    from_addr: str
    to_addr: str
    token: str                          # USDC ...
    amount: Decimal = Field(ge=0)
    block_number: int
    event_ts: datetime

class MacroSignal(BaseModel):
    """OpenBB 宏观/基本面信号。"""
    indicator: str                      # CPI / UNRATE / ...
    value: Decimal
    released_at: datetime
    provider: str                       # fred / bls / sec

class FairValueEstimate(BaseModel):
    """信号层产出：某市场的公允价/公允概率。"""
    condition_id: str
    fair_prob: Decimal = Field(ge=0, le=1)
    conf_lo: Decimal = Field(ge=0, le=1)
    conf_hi: Decimal = Field(ge=0, le=1)
    basis: str                          # latency_arb / structural / macro
    inputs_ref: list[str]               # 可追溯的输入事件 id
    ts: datetime

class ArbSignal(BaseModel):
    """信号层产出：可执行的套利价差信号（只读阶段的最终产物）。"""
    strategy: str                                # single/multi/semantic/latency/time_decay/copy
    condition_id: str
    side: Side
    market_prob: Decimal = Field(ge=0, le=1)   # p
    fair_prob: Decimal = Field(ge=0, le=1)      # q
    edge: Decimal                                # q-p（YES）或 p-q（NO）
    edge_after_cost: Decimal                     # 扣费/滑点/Gas 后净边际
    confidence: Decimal = Field(ge=0, le=1)
    legs: list[dict] = []                        # 多腿策略（①②③）的各腿 token/价/量
    exit_liquidity_ok: bool = True               # 出场流动性预检（§6.2）
    wash_suspect: bool = False                   # 命中洗盘检测则为 True（§6.1）
    ts: datetime

class MarketGroup(BaseModel):
    """互斥市场组（策略② 多条件套利）。来源 Gamma API 事件分组。"""
    event_id: str
    condition_ids: list[str]                     # 组内全部互斥结果
    yes_sum: Decimal                             # ΣYesᵢ
    is_exhaustive: bool                          # 是否穷尽（漏项=伪套利）
    is_mutually_exclusive: bool

class SemanticLink(BaseModel):
    """跨市场语义等价映射（策略③）。"""
    primary_condition_id: str
    equivalent_basket: list[str]                 # 等价子市场组合
    equivalence_conf: Decimal = Field(ge=0, le=1) # LLM 等价置信，低于阈值不交易
    verified_by: str                             # llm / rule / human（高风险需二次确认）

class ExpiringTarget(BaseModel):
    """临到期极端目标（策略⑤ 时间衰减卖 No，高风险）。"""
    condition_id: str
    end_date: datetime
    days_to_expiry: int
    no_prob: Decimal = Field(ge=0, le=1)         # 当前 No 价（目标区间 0.92–0.98）
    is_extreme_target: bool                      # 极端目标识别
    settlement_rule: str                         # 结算规则解析（语义模糊→人工）

class WalletProfile(BaseModel):
    """链上钱包画像（策略⑥ 聪明钱跟单，仅公开数据，涉合规）。"""
    address: str
    trades: int
    win_rate: Decimal = Field(ge=0, le=1)
    realized_pnl: Decimal
    first_active: datetime                       # "新活跃且高胜率"识别
    insider_suspect: bool                        # 疑似内幕→强制人工合规复核（§9.7）
```

## 5. 延时套利锚点：计算口径

目标：用 Binance 短周期价格推导加密类预测市场（如"BTC 在某时点 > X"）的公允概率，与 Polymarket
隐含概率比对。

1. **公允价/公允概率 `q`**：
   - 对"阈值型"市场（BTC > X by T），由 Binance 现货/合约价 + 隐含波动/历史波动估计到期超过 X 的概率
     （如对数正态近似或经验分布），输出 `FairValueEstimate.fair_prob` 与置信区间。
2. **市价 `p`**：取 Polymarket `OrderBookSnapshot.mid`（或最优买卖加权），即市场隐含概率。
3. **边际**：`edge = q − p`（买 YES）或 `p − q`（买 NO）；`edge_after_cost = edge − (fee + slippage + gas_bps)`。
4. **时间对齐（关键）**：
   - 仅当 `|polymarket.recv_ts − binance.recv_ts| ≤ Δ_max`（如 ≤ 250ms，配置项）才生成信号，
     否则标记 `stale` 不输出。
   - 用 `recv_ts`（单调时钟）做对齐，`event_ts` 仅作参考，规避数据源时钟漂移。
5. **触发**：`edge_after_cost ≥ θ` 且置信区间不跨越 `p` → 产出 `ArbSignal`。

## 6. 其他策略信号口径（②③⑤⑥ + 防护信号）

> 对应主文档 §1.2 策略族。所有信号输出统一收敛到 `ArbSignal`（带 `strategy` 标签），
> 并必须通过 §6.1 洗盘检测与 §6.2 流动性预检后才置 `wash_suspect=False / exit_liquidity_ok=True`。

### 6.0 结构与互斥/语义/时间衰减/跟单信号
- **① 单条件结构（`structural.py`）**：单市场 `bestAsk_yes + bestAsk_no < 1 − 成本` → 双买套利；
  输出 `ArbSignal(strategy="single", legs=[yes,no])`。
- **② 多条件互斥（`multi_condition.py`）**：拉取事件互斥组（`MarketGroup`），校验 `is_exhaustive &&
  is_mutually_exclusive`；`yes_sum < 1 − 成本` → 覆盖全组。**穷尽性/互斥性不成立则不产信号**（防伪套利）。
- **③ 跨市场语义（`semantic_map.py`）**：LLM 产出 `SemanticLink`，`equivalence_conf ≥ 阈值` 且
  `verified_by ∈ {rule, human}`（高风险，纯 LLM 不放行）才比对等价组合与主市场价差。
- **⑤ 时间衰减（`time_decay.py`）**：筛 `days_to_expiry ≤ 5 && is_extreme_target && no_prob ∈[0.92,0.98]`，
  且 `settlement_rule` 非模糊（模糊→人工）。产 `ArbSignal(strategy="time_decay")`，**标记尾部风险等级**供风控。
- **⑥ 聪明钱跟单（`smart_money.py`）**：由链上构建 `WalletProfile`，筛 `win_rate ≥ 阈值 && trades 充足`；
  `insider_suspect=True` → **不自动产信号，转人工合规**（§9.7 主文档）。仅用公开数据，无身份关联/规避机制。

### 6.1 洗盘/自成交检测（`wash_filter.py`，全策略前置）
- 启发式：买卖对手高度重叠的地址环、极端价位（如 0.036↔0.98）大额自挂自接、深度与真实成交背离、
  关联地址簇。命中 → `wash_suspect=True`，该市场**剔除出可交易集并告警**。
- 目的：对冲来源稿提到的 **Data Inflation / 3900 万机会虚高**（主文档 §0）。

### 6.2 出场流动性预检（`liquidity.py`，开仓前置）
- 估算在目标持有期/结算前能以可接受滑点平掉仓位所需的对手盘深度；不足 → `exit_liquidity_ok=False`，
  **不开仓**（防流动性陷阱，冷门市场尤甚）。

## 7. 第一阶段最小闭环（验收基准）

**范围**：Polymarket + Binance 两路采集 → Schema 校验 → 价差信号输出（**只读，不下单**）。

最小闭环组件：
- `collectors/polymarket.py`：订阅目标市场订单簿 WS，产出 `OrderBookSnapshot`。
- `collectors/binance.py`：订阅 BTC/ETH WS，产出 `PriceTick`。
- `signals/fair_value.py` + `signals/latency_arb.py`：产出 `FairValueEstimate` 与 `ArbSignal`（策略④）。
- `signals/structural.py`：单条件 `YES+NO<1` 结构信号（策略①，仅需 Polymarket 订单簿、风险最低，
  纳入 P1 只读输出）；同时接入 `wash_filter.py` 与 `liquidity.py` 作为前置过滤。
- 输出端：结构化日志 + 落库 + Lightweight Charts 仪表盘（信号频率、价差曲线）。

**验收标准（P1）**：
- [ ] 两路数据稳定接入，断线自动重连，`health()` 可观测。
- [ ] 所有进入信号层的数据 100% 经过 pydantic 校验；注入非法数据时被拒绝并告警（含单元测试）。
- [ ] 能对至少 1 个加密类市场持续产出 `ArbSignal`，含 `edge_after_cost` 与时间对齐判定。
- [ ] 仪表盘实时显示价差/信号频率；日志可完整回溯每条信号的输入事件 id。
- [ ] 全程不存在任何下单/签名调用路径（只读保证）。

## 8. 测试策略

| 类型 | 重点 |
| --- | --- |
| 单元测试 | Schema 校验（合法/非法/边界，价格 ∈[0,1]、非负、枚举）；公允价计算口径；时间对齐判定 |
| 集成测试 | collector 用录制的 WS/REST 回放数据驱动；端到端产出 ArbSignal |
| 契约测试 | 真实 API 返回结构变更时，校验层能识别并告警（防止上游 schema 漂移静默通过） |
| 韧性测试 | 断线重连、乱序/迟到事件、时钟漂移、限流 |
| 策略正确性 | 互斥组穷尽/互斥校验（漏项=伪套利须被拒）；语义等价非纯 LLM 放行；洗盘检测命中样本；出场流动性不足拒单；时间衰减尾部标记 |

> 遵循项目测试要求：核心校验与信号计算覆盖率 ≥ 80%，先写测试（RED）再实现（GREEN）。
